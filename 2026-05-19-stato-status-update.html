How each number is fetched, compared, displayed data flow

For full transparency — every tile and chart on Stato has the same 3-step shape: (1) pull from upstream, (2) store/cache locally, (3) render in the UI. Here's how that runs per source so when you spot a discrepancy you know exactly where to look.

LeadByte (leads + revenue per campaign)

Pull: hourly cron + on-demand "Sync now" hits /reports/leadactivity (daily lead counts), /reports/campaign (windowed totals incl. revenue + payout), /reports/buyer (caps + buyer breakdown), and /deliveries (individual lead rows).
Store: lead_deliveries table (one row per campaign per day) + client_campaigns junction. Daily revenue is pro-rated from windowed totals (leadCount × windowRevenue / windowLeads) so the sum across days = the LeadByte invoice total.
Show: dashboard Leads tile, Revenue Overview chart, per-campaign Revenue vs Cost chart, buyer portal lead tables.
Verify: open LeadByte UI for the same campaign + window. Stato should match the leads-count exactly and the revenue to within rounding (we round to 2dp on the per-day pro-rate).

Catchr (ad spend per platform + account)

Pull: daily cron via the Catchr MCP — for each connected account (Facebook, Google Ads, TikTok, Taboola) we call run_api_request_json with the platform's normalised date field + spend metric.
Store: ad_spend table — one row per platform/account/date. Dead accounts (CUSTOMER_NOT_ENABLED) are tracked in-process and skipped after the first failure for the rest of the run.
Show: dashboard Cost tile, per-campaign Cost column (sums every account_id linked via Traffic Sources for the last 30 days), Performance-by-source widget, Profit / Margin derivations.
Verify: open Catchr UI → pick the same platform + account + date range → totals should match exactly. The Catchr dashboard banner ("9 of 15 accounts working") is the single source of truth for which accounts are actually flowing.

Xero (revenue + invoices + bank + VAT)

Pull: Custom Connection (server-to-server OAuth, no consent flow). Hourly global invoice sync at :15 iterates every non-churned client. Bank widget calls /Accounts filtered to BANK type, falls back to /Reports/BankSummary if zero. VAT widget calls /Reports/VATQuarters. 429 retries with backoff + stale-while-revalidate so the UI never shows "fetch failed".
Store: invoices table mirrors Xero state (status, totals, due date). Bank balance + VAT are cached in-memory with TTL (60min for VAT, 15min for bank).
Show: dashboard Outstanding/Overdue tile, Bank Accounts tile, VAT Liability tile, /finance/invoices grid, per-client Invoices tab.
Verify: open Xero → Business → Invoices (filter same status) → sum should match Stato. The new /api/v1/integrations/xero/health endpoint gives a machine-readable diff per probe.
