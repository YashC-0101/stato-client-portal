What shipped today 10 major fixes

Each line below is something a careful auditor would have called out as "looks wrong" yesterday and "looks honest" today.

1. Dashboard time-range filter

Dropdown top-right on /dashboard: Last 7d · This month · Last month · 90d · 6m · 12m. Drives all 5 value tiles (Revenue, Cost, Profit, Margin, Leads), the Revenue Overview area chart, the Invoice Status bar chart, and the Campaign Sources pie. 2. Profit / Margin no longer lies

Was: -1924% margin because lifetime revenue was being divided by only 2 months of Catchr cost. Now: revenue and cost are pulled over matching windows (365-day rolling revenue, 90-day rolling cost) so the ratio is honest. Today's number across the business: £733,172 revenue · £763,662 cost · -£30,490 profit · -4.2% margin. That's what a real lead-gen agency's P&L looks like when you stop averaging away the gaps.

3. lead_deliveries.revenue writer fixed — 4,185 rows backfilled

Until today the writer was hardcoding revenue: 0 on every lead-delivery row, so the daily-revenue chart on every campaign page was a flat line at zero. Fixed by pro-rating from LeadByte's /reports/campaign windowed totals down to per-day numbers (leadCount × (windowTotal / windowLeadsSum)). Added a no-overwrite guard so a future failed sync can never blank out a previously-populated row. £70k+ of revenue is now visible on the Revenue vs Cost campaign chart that used to render empty.

4. Catchr ad spend now folded into per-campaign Profit / Margin

On /campaigns/<id> the header used to compute Profit as revenue - leadbyte_cost only, completely ignoring the Facebook/Google/TikTok ad spend driving the leads. Result: every campaign appeared 100% margin. Fixed: backend now joins each campaign's Traffic Sources to Catchr's ad_spend table (any account that's linked, over the last 30 days) and adds that to total cost. Solar Panels detail now reads -£253,398 / -200.2% — the real number; the campaign list page sees the same fold-in.

5. Multi-account-per-source for Traffic Sources

Before: each Traffic Source row could link to exactly ONE Catchr ad account. Sam's Solar campaign actually has 5 Catchr accounts running (Solar Incentives, TheSolarGeeks, Solar Discounts, MYSOLAR, Solar Compare UK) and the source was linked to Solar Compare UK which is genuinely empty in Catchr — so the row showed £0 and you (rightly) asked "where's the spend?". Fixed: each Traffic Source row now carries a JSONB account_ids[] array; the Catchr account picker lets you tick multiple at once. Solar Panels' traffic source is now linked to all 5 → rolls up to £379,979.82 over 30 days.

6. Catchr field-map bugs in 3 platforms

Facebook was silently writing 0 rows because the date_start field was being dropped — switched to Catchr's normalised date field (_NORMALIZED_DATE_FIELD_YEAR_MONTH_DAY). Taboola was throwing ON CONFLICT cannot affect row a second time because campaignId was returning empty → fixed the field name (campaign not campaign_id) AND added within-batch dedupe so duplicate keys from upstream can never crash the writer. Google Ads dead-account skip: any account returning CUSTOMER_NOT_ENABLED is now tracked in an in-process Set and skipped for the rest of the run instead of failing every account-day pull.
