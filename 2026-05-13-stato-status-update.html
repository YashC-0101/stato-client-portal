Tasks revamp live

Your Loom said: "the tasks are very basic, there's nothing in here." Every gap from that walkthrough is now addressed.

✅ Subtasks — checkbox items inside any task

Open a task → Subtasks card. Type "review draft", press +, repeat for each step. Click the checkbox to mark a subtask done — it strikes through and the counter at the top updates (e.g. "3/5 done"). Delete with the trash icon.

Every subtask change writes to the Activity feed so you can see who completed what and when.

✅ Attachments — PDF / image / any file

Open a task → Attachments card → Upload. Files go to the same Cloudflare R2 bucket the client documents use, so your storage is unified. Each row shows the filename, size, and timestamp. Click the download icon to open it in a new tab (fresh-signed URL every time — no static links leaking).

Useful for: pasting a screenshot of a bug, attaching a PDF the task references, saving an email export against the task.

✅ Activity feed — timeline of everything that happened

Every action on a task — created, status changed, comment added, subtask completed, attachment uploaded, time block updated, parent changed, recurrence set — is logged with the actor's name and timestamp. Shown as a left-rail timeline on the detail page.

Distinct from the old audit log — that's still there in the database but the Activity feed is the user-facing version.

✅ Time blocks — 15m / 30m / 1h / 2h / Half day / Full day

Your Loom: "I should be able to say this is a one hour task, this is a two hour task." Done.

On the create form: Time block dropdown with the buckets above. On the detail page: visible in the Relationships card and editable inline. On the list page: new Time column with a ⏱ badge. On the kanban board: same badge on every card.

Coarse buckets on purpose — free-form minutes input was the original plan but most ops tasks fall into these 6 chunks and the picker is much faster to use than typing a number.

✅ Linked SOP — pick an SOP the task references

Each task can link to one published SOP. Useful for: "Process weekly Xero export" → links to the actual SOP doc explaining the steps. On the create form: Linked SOP dropdown (published SOPs only — drafts not selectable). On the detail page: shows the SOP title as a clickable link in the Relationships card, editable inline.

✅ Parent task — group sub-tasks under a project

Lightweight project structure without building a full Projects entity. Any task can be the parent of any other task.

  • On the create form: Parent task dropdown.
  • On the detail page: Children card lists all children, with their status + priority badges. "Add child" button on that card pre-fills the parent for you.
  • Click a child → opens its detail page. The Relationships card on the child shows the parent as a clickable link.
✅ Recurring tasks — cron schedule (daily / weekly / monthly)

Set a recurrence pattern on a task (e.g. "every Monday at 9am" = 0 9 * * 1) and a background worker clones the task as a fresh todo on every scheduled fire. The worker ticks every 5 minutes.

Two safeguards:

  • Clones don't recur themselves — only the original schedule fires, so you never get an exponentiating tree of duplicates.
  • Bad cron expressions are quarantined — the worker detects invalid syntax, clears the next-run timestamp, and logs a warning so the row doesn't get retried every 5 minutes forever.

UI for picking schedules is via API for now — full picker is on the polish list.

Top-right of the tasks page: a toggle that switches between My tasks (filtered to your name or email) and All tasks (no filter — see everyone). Your choice persists across reloads — each team member can pin their preferred default.

How it works — clicking the SOS button

2. Modal pops up with a textarea ("what's going on?").

PDF editor (DocuSign-style) live

Your Loom asked for drag-place signature fields like DocuSign. Live now on the agreement send flow.

🖊️ The editor flow

On /agreements → click "Send for signature" → fill in client + signer + upload the PDF → instead of clicking "Send envelope" directly, click the new "Place fields…" button next to it.

You land on /agreements/editor:

  1. The full PDF renders on screen, all pages
  2. Toolbar at the top with three field types: Signature (blue), Date signed (amber), Text (neutral)
  3. Click a tool button → it arms. Click anywhere on the PDF → a coloured box drops at that point
  4. Drag any placed box to reposition. Click the ✕ to delete
  5. Place as many fields as you want (cap 50 per document)
  6. Click "Send for signature" at the top → the PDF goes to SignNow with the fields pre-positioned. Your signer sees a clean document with ONLY the boxes you placed, in the positions you placed them

Backward-compatible: the old "send envelope" button still works for free-form invites (signer places signature wherever) — your existing workflow doesn't break.
