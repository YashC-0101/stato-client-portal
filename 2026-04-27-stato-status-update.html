Done What we delivered today

1 · LeadByte integration went live

Wired your LeadByte API key into the backend. 28 real campaigns from clinical.leadbyte.co.uk are now syncing every 2 minutes — Solar Panels UK, Hearing Aids (IE/CH/PL), Will Writing, Lasting Power of Attorney, Conservatory Upgrades, and 23 more. Last-month report shows real revenue (Hearing Aids PL: £38,850 across 3,380 leads).

Three pre-existing client-side bugs surfaced once real data flowed and were fixed (report-envelope shape, campaign-as-object refs, currency normalisation). The client now also surfaces real API errors instead of silently falling back to mock data — so any future credential issue is visible immediately.

2 · Cloudflare R2 file storage went live

stato-production bucket created with an Object-scoped API token. Backend client uses AWS SDK v3 with a Cloudflare-compatibility tweak (auto-checksum disabled — caused signature mismatches on browser uploads). CORS configured to allow the live and staging frontend origins.

Five upload folders are wired across the app:

  • Client documents — Clients → any client → "Documents" tab
  • Invoice attachments — Invoices → any invoice → "Attachments" card (receipts, proof of payment)
  • Staff documents — Staff → "Documents" tab → pick member (contracts, NDAs, payslips)
  • Campaign creatives — Campaigns → any campaign → "Creatives" card (ad images, videos, copy). These show up automatically in the client's portal Compliance tab.
  • Agreement source PDFs — the SignNow Send dialog now uploads to R2 first, lifting the previous 10 MB API body cap to 50 MB.

3 · Production deployed to Railway + Vercel

Backend on Railway: https://sato-backend-production.up.railway.app Auto-rebuilds on every push to main.

Frontend on Vercel: https://sato-frontend.vercel.app

PostgreSQL: 26 tables migrated (0000 → 0006), 4 internal users seeded plus the demo client Apex Media Ltd. Redis live for BullMQ jobs. JWT secrets generated to production strength, CORS allow-list multi-origin, refresh-on-page works (SPA rewrite added this evening).

4 · Workflow engine wired to real Xero / Resend

Two BullMQ handlers now do real work, not stubs:

  • Auto-invoice — for each active weekly_auto client, sums lead deliveries from the last 7 days, builds a single-line invoice, creates the row in the DB, and pushes a draft ACCREC invoice to Xero. Per-client errors don't fail the batch.
  • Monthly-validated — for each monthly_validated client, sums lead deliveries for the prior calendar month, builds an HTML revenue table, queues an email to owner@stato.app via Resend so you can request sign-off before invoicing.

Both are reachable from the Workflows page → "Run Now". The chase-overdue cron job continues to run daily at 9:00 UTC.

5 · Code health

Backend: 387 / 387 tests passing, 0 TypeScript errors. Frontend: 88 / 88 tests passing. 12 commits pushed to main across both repos this evening, all promoted via the BRANCHING.md Development → main workflow we documented.
